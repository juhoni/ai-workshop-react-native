How to Use AI in IDEs

Plan Mode

Ask AI to plan steps without executing

When: Complex features, architecture decisions, unclear requirements

Ask Mode

Questions without code changes

When: Understanding code, exploring APIs, debugging

Agent Mode

AI makes changes autonomously

When: Repetitive edits, boilerplate, clear requirements

Review Changes

Always check diffs before accepting

  • Verify logic is correct
  • Check for security issues
  • Test TypeScript types

Undo Changes

Git is your safety net

Use git reset, revert, or IDE undo. Never accept changes blindly.

Changing Models

Different tasks need different models

Fast models: Quick edits. Smart models: Complex reasoning.

Providing Context to AI

Tag Files

Use @filename or #file:path to reference files

Best for: Related components, configs, type definitions

Select Code

Highlight code in editor before asking

Best for: Specific functions, bugs, refactoring a section

Provide Project Docs

Share READMEs, guidelines, architecture docs

Up-to-date docs give AI guardrails for code style, patterns, and feature requirements

Add Error Messages

Full stack traces, not just titles

Include platform (iOS/Android) and device info

Provide URLs

Link to docs for AI to browse

AI training data is frozen — help it access current info

DON'T OVERLOAD

Provide only relevant context. Too much fills the working context window and wastes tokens.

Part 3

The Pitfalls

Critical warnings and anti-patterns

Pitfall #1

Hallucinated APIs

AI confidently suggests props and methods that simply do not exist.

EXAMPLE

preloadDistance={500}

This prop does not exist!

ALWAYS VERIFY:

  • Check official docs
  • Use TypeScript autocomplete
  • Test on both platforms

Pitfall #2

Outdated Patterns

AI Often Suggests:

// Class components

class MyComponent extends...

// Old Reanimated v2

Animated.timing().start()

You Should Use:

// Functional + hooks

function MyComponent() {}

// Reanimated v4

x.value = withTiming(100)

More Critical Pitfalls

Copy-Paste Syndrome

Code without understanding = tech debt

Security Blindspots

AsyncStorage instead of SecureStore

Over-Engineering

AI loves unnecessary abstractions

Version Blindness

AI doesn't know your RN 0.81, Expo 54

Part 4

Effective Prompting

Getting better results from AI

Good Prompts vs. Bad Prompts

BAD: Vague

"Fix this navigation crash"

"Make animation smoother"

Result: Generic, often wrong answers

GOOD: Specific

"Using expo-router 6.0, crash on Android with [error]. Code: [snippet]"

"Reanimated 4.1, spring scale on press. Target: 60fps Android"

Result: Targeted, working solutions

Key Prompting Strategies

1. Specify Versions

"Using RN 0.81, Expo 54..."

2. Provide Context

Errors, code, constraints

3. Ask for Explanations

"Why does this work?"

4. Iterate

Follow-ups, request changes

5. Verify Output

Docs, TypeScript, test

AI = helpful colleague, not oracle
